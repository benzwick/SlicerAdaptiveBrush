# Screenshots

This folder contains screenshots for the Extension Index and README.

## How Screenshots Are Generated

Screenshots are generated through **test-driven documentation**:

1. **Test cases** in `SegmentEditorAdaptiveBrushTester/TestCases/test_docs_*.py` capture screenshots with `doc_tags`
2. **Tests workflow** (`.github/workflows/tests.yml`) runs these tests in Slicer
3. **Screenshot extraction** copies tagged screenshots to `docs/source/_static/screenshots/`
4. **Docs build** includes screenshots in generated documentation
5. **Docs workflow** deploys to GitHub Pages

## Screenshot Sources

| Location | Purpose |
|----------|---------|
| `test_docs_algorithms.py` | Algorithm panel screenshots |
| `test_docs_ui_reference.py` | UI element screenshots |
| `test_docs_workflows.py` | Tutorial workflow screenshots |
| `test_docs_reviewer.py` | Reviewer module screenshots |

## Manual Screenshot Capture

For local development or README updates:

```bash
# Capture screenshots locally
Slicer --python-script scripts/capture_screenshots.py --exit

# Or interactively
Slicer --python-script scripts/capture_screenshots.py
```

## Extension Index URL

For the 3D Slicer Extension Index CMakeLists.txt:

```cmake
set(EXTENSION_SCREENSHOTURLS
  "https://raw.githubusercontent.com/benzwick/SlicerAdaptiveBrush/main/Screenshots/main-ui.png"
)
```

## Screenshot Specifications

- **Format:** PNG
- **Resolution:** 1920x1080 (from Xvfb virtual display in CI)
- **Sample Data:** MRHead or MRBrainTumor1 (bundled with Slicer)
- **Layout:** Conventional view (Red, Yellow, Green slice views + 3D)

## Manifest

The `manifest.json` file (generated by capture script) contains metadata:
- Filename and description
- Capture type and dimensions
- Generation timestamp
